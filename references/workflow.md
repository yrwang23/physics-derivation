# 推导工作流（中文）

所有对用户的输出必须使用中文。

## 核心原则

1. **推导是主线，确认知识自然嵌入其中** — 在推导的每一步中，当需要某个概念或方法时停下来询问用户
2. **不假设用户掌握任何概念** — 参考材料库仅用于 agent 查阅公式定义，使用前必须询问
3. **个人知识库初始为空，按分类存放** — 通过对话逐步填充
4. **展示"作用"优先于"定义"** — 先说明方法在当前步骤中的角色，再询问用户是否了解

---

## 完整工作流

### 第 1 步：理解用户的请求

确认用户想推导的公式及其上下文。用中文回复。

### 第 2 步：开始逐步骤推导

从已知知识出发，逐步推导目标公式。每到一个需要新概念或新方法的步骤：

**第 2a 步：展示该方法在当前步骤中的作用**
用一两句话说清这个方法在这里怎么用、发挥了什么作用。

**第 2b 步：询问用户是否了解**

**第 2c 步：根据用户回答分叉处理**

- 用户回答"了解/会"：
  1. 询问用户："这个知识点属于哪个分类？（量子多体 / 量子信息与神经网络 / 凝聚态计算 / 其他）"
  2. 将该概念加入个人知识库对应分类目录下
  3. 更新 INDEX.md 中对应分类的表格
  4. 继续推导

- 用户回答"不了解/不会"：
  1. 用中文讲解概念的核心内容
  2. 确认理解后，同样询问分类并入库
  3. 继续推导

**第 2d 步：涉及文献时**
- 显示文献信息，询问用户能否提供 PDF
- 能 → 保存到 papers/
- 不能 → 基于 agent 自身知识继续

### 第 3 步：继续推导

重复第 2 步的过程，直到得到目标公式。

### 第 4 步：生成渲染视图（KaTeX 网页）

推导完成后：
1. 将整个推导整理为 markdown 文件（含 LaTeX 公式）
2. 运行 `scripts/render_katex.py` 生成 KaTeX 渲染的 HTML
3. 用 in-app browser 打开该 HTML

### 第 5 步：迭代问答

用户追问时回到第 2 步处理新知识点。

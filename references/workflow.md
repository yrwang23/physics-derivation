# 推导工作流（中文）

所有对用户的输出必须使用中文，公式使用 Unicode 符号 + 简单文本格式。

## 核心原则

1. **推导是主线，确认知识自然嵌入其中** — 在推导的每一步中，当需要某个概念或方法时停下来询问用户
2. **不假设用户掌握任何概念** — 参考材料库仅用于 agent 查阅公式定义，使用前必须询问
3. **个人知识库初始为空，按分类存放** — 通过对话逐步填充
4. **展示"作用"优先于"定义"** — 先说明方法在当前步骤中的角色，再询问用户是否了解

## ## 公式书写规范

Chat 对话和最终 .md 文档**统一使用相同格式**：
- **行内公式**用 \(...\) 包裹
- **独立行公式**用 $$...$$ 包裹

### Chat 对话示例
- 薛定谔方程: \(i\hbar \partial_t |\psi\rangle = \hat{H}|\psi\rangle\)
- 费米子反对易: \(\{c_i, c_j^\dagger\} = \delta_{ij}\)
- 色散关系: \(\omega = \sqrt{4\beta/m}\,|\sin(ka/2)|\)

### 最终 .md 文档
按 `derivation-template.md` 输出，格式同上。

## 完整工作流

### 第 1 步：理解用户的请求

确认用户想推导的公式及其上下文。用中文回复。

### 第 2 步：开始逐步骤推导（如果用户上传了PDF，并指定推导PDF当中的某个公式，优先考虑第2d步）

从已知知识出发，逐步推导目标公式。每到一个需要新概念或新方法的步骤：

**第 2a 步：展示该方法在当前步骤中的作用**
用一两句话说清这个方法在这里怎么用。

**第 2b 步：询问用户是否了解**

**第 2c 步：根据用户回答分叉处理**

- 用户回答"了解/会":
  1. 询问用户："这个知识点属于哪个分类？（量子多体 / 量子信息与神经网络 / 凝聚态计算 / 其他）"，并询问用户是否需要将这个知识点加入个人知识库，如果用户回答需要，执行下面的2，3操作，如果不需要就继续推导
  2. 将该概念加入个人知识库对应分类目录下
  3. 更新 INDEX.md 中对应分类的表格
  4. 继续推导

- 用户回答"不了解/不会":
  1. 用中文讲解概念的核心内容
  2. 确认理解后，同样询问分类并是否入库
  3. 继续推导

**第 2d 步：涉及文献时**
- 根据用户提供的文献，对被提问的公式的推导流程进行溯源，显示推导该公式需要涉及到的相关文献信息，并自己去查找这些文献，如果查找失败，询问用户能否提供溯源文献的 PDF
- 能 → 保存到 papers/
- 不能 → 基于 agent 自身知识继续
- 有了参考文献以后，基于参考文献的推导流程，重复上述2b 2c的过程，根据用户的知识库需要询问用户是否了解该知识点

### 第 3 步：继续推导

重复第 2 步的过程，直到得到目标公式。

### 第 4 步：生成完整推导总结

当所有前置概念确认完毕、推导完成后，询问用户是否要一份完整的推导文档。

如果需要，按 `derivation-template.md` 的格式输出，注意：
- **全部使用中文**
- **公式使用标准 LaTeX markdown 格式**（$...$ / $$...$$），可被 Typora、VS Code 等渲染器编译
- **Chat 对话和最终文档统一使用 LaTeX 格式**
- 每步标注依据的知识库条目名称
- 包含关键假设与近似一览

### 第 5 步：迭代问答

用户追问时回到第 2 步处理新知识点。
